# TurboGitHub 测试指南

## 🧪 系统托盘窗口恢复测试

### 问题描述
点击系统托盘图标菜单的"显示窗体"后，窗体未能成功恢复

### 修复内容
- 增强托盘事件处理逻辑
- 添加窗口大小恢复命令
- 添加强制重绘机制
- 添加窗口聚焦命令

---

## 📋 测试步骤

### 准备工作

1. **编译项目**
   ```bash
   # 调试版本（快速）
   cargo build -p turbogithub-gui
   
   # 或 Release 版本（慢速，但更接近最终版本）
   cargo build -p turbogithub-gui --release
   ```

2. **启动应用程序**
   ```bash
   # 使用启动脚本
   .\启动 TurboGitHub.bat
   
   # 或直接运行
   .\target\debug\turbogithub-gui.exe
   # 或
   .\target\release\turbogithub-gui.exe
   ```

---

### 测试用例 1: 基本窗口恢复

**步骤**:
1. 启动 TurboGitHub 应用程序
2. 等待窗口完全加载（约 3-5 秒）
3. 点击窗口右上角的关闭按钮（X）
4. 窗口应隐藏到系统托盘
5. 等待 5 秒
6. 右键点击系统托盘中的 TurboGitHub 图标
7. 选择"显示窗口"
