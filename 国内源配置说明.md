## 📁 配置文件位置

### 项目级配置
```

```
此配置仅对 TurboGitHub 项目生效。

### 全局配置（可选）
```
~/.cargo/config.toml
```
此配置对所有 Rust 项目生效。

## 🚀 快速开始

### 方法 1：使用项目配置（推荐）

项目已包含 `.cargo/config.toml`，直接使用即可：

```bash
# 编译项目（自动使用国内源）
cargo build

# 运行项目
cargo run

# 更新依赖
cargo update
```

### 方法 2：配置全局镜像源

运行配置脚本：

```powershell
.\setup-rust-mirror.ps1
```

或手动创建 `~/.cargo/config.toml`：

```toml
[source.crates-io]
replace-with = 'ustc'

[source.ustc]
registry = "sparse+https://mirrors.ustc.edu.cn/crates.io-index/"

[net]
git-fetch-with-cli = true
```

## 🛠️ 故障排除

### Q1: 下载速度慢

**解决方案**：
1. 确认 `.cargo/config.toml` 存在
2. 检查镜像源配置
3. 切换到其他镜像源：
   ```toml
   [source.crates-io]
   replace-with = 'tuna'  # 使用清华源
   ```

### Q2: 依赖下载失败

**解决方案**：
```bash
# 清理缓存
cargo clean

# 强制更新
cargo update -Z direct-minimal-versions

# 重新下载
cargo fetch
```

### Q3: Git 依赖克隆失败

**解决方案**：
```bash
# 配置 Git 使用国内镜像
git config --global url."https://mirrors.ustc.edu.cn/github.com/".insteadOf "https://github.com/"

# 或者使用代理
git config --global http.proxy http://127.0.0.1:7890
```

### Q4: 证书验证失败

**解决方案**：
```bash
# 临时禁用证书验证（不推荐）
$env:CARGO_NET_GIT_FETCH_WITH_CLI = "true"

# 或者更新证书
rustup self update
```
