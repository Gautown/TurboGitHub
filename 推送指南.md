## 🌐 网络连接问题解决方案

### 问题症状
```
fatal: unable to access 'https://github.com/Gautown/TurboGitHub.git/':
Failed to connect to github.com port 443 after 21072 ms:
Could not connect to server
```

### 解决方案

#### 1. 使用 GitHub 加速器

**推荐工具**:
- [FastGithub](https://github.com/dotnetcore/FastGithub)
- [Watt Toolkit](https://github.com/BeyondDimension/SteamTools)
- [GitHub520](https://github.com/dotnetcore/GitHub520)

**FastGithub 使用**:
```bash
# 下载并运行 FastGithub
FastGithub.exe

# 然后推送
git push TurboGitHub main
```

#### 2. 修改 hosts 文件

**步骤**:
1. 以管理员身份打开记事本
2. 打开文件：`C:\Windows\System32\drivers\etc\hosts`
3. 添加以下内容:
```
140.82.114.4 github.com
140.82.112.5 api.github.com
185.199.108.153 assets-cdn.github.com
185.199.109.153 raw.githubusercontent.com
185.199.110.153 gist.githubusercontent.com
```
4. 保存文件
5. 刷新 DNS: `ipconfig /flushdns`
6. 重试推送

#### 3. 使用代理

**配置 HTTP 代理**:
```bash
# 设置代理（替换为你的代理地址）
git config --global http.proxy http://127.0.0.1:7890
git config --global https.proxy https://127.0.0.1:7890

# 推送
git push TurboGitHub main

# 取消代理（推送后）
git config --global --unset http.proxy
git config --global --unset https.proxy
```

**使用 TurboGitHub 自身加速**:
```bash
# 启动 TurboGitHub
.\启动 TurboGitHub.bat

# 配置 Git 使用代理
git config --global http.proxy http://127.0.0.1:10808
git config --global https.proxy http://127.0.0.1:10808

# 推送
git push TurboGitHub main

# 取消代理
git config --global --unset http.proxy
git config --global --unset https.proxy
```

#### 4. 使用 Git 协议

```bash
# 修改远程 URL 为 Git 协议
git remote set-url TurboGitHub git://github.com/Gautown/TurboGitHub.git

# 推送
git push TurboGitHub main

# 改回 HTTPS（可选）
git remote set-url TurboGitHub https://github.com/Gautown/TurboGitHub.git
```

#### 5. 增加 Git 缓冲区

```bash
# 已配置（之前已执行）
git config --global http.postBuffer 524288000
git config --global https.postBuffer 524288000

# 推送
git push TurboGitHub main
```

#### 6. 使用 SSH 连接

**步骤**:
1. 生成 SSH 密钥:
```bash
ssh-keygen -t ed25519 -C "your_email@example.com"
```

2. 添加公钥到 GitHub:
   - 复制公钥：`cat ~/.ssh/id_ed25519.pub | clip`
   - 打开 GitHub → Settings → SSH and GPG keys
   - 点击 New SSH key
   - 粘贴公钥

3. 修改远程 URL:
```bash
git remote set-url TurboGitHub git@github.com:Gautown/TurboGitHub.git
```

4. 推送:
```bash
git push TurboGitHub main
```

## 🛠️ 故障排除

### Q1: 推送失败 - Connection reset

**原因**: 网络连接被重置

**解决方案**:
```bash
# 使用加速器
FastGithub.exe

# 或使用代理
git config --global http.proxy http://127.0.0.1:7890
git push TurboGitHub main
```

### Q2: 推送失败 - Authentication failed

**原因**: 身份验证失败

**解决方案**:
```bash
# 清除缓存的凭证
git credential-manager erase

# 重新推送（会提示输入凭证）
git push TurboGitHub main
```

**或使用 Personal Access Token**:
1. 访问 GitHub → Settings → Developer settings → Personal access tokens
2. 生成新 token（勾选 repo 权限）
3. 使用 token 作为密码推送

### Q3: 推送失败 - Large files

**原因**: 文件过大

**解决方案**:
```bash
# 安装 Git LFS
git lfs install

# 跟踪大文件
git lfs track "*.psd"
git lfs track "*.zip"

# 重新添加并提交
git add .gitattributes
git commit -m "chore: 配置 Git LFS"

# 推送
git push TurboGitHub main
```

### Q4: 推送缓慢

**原因**: 网络速度慢

**解决方案**:
```bash
# 使用压缩
git config --global core.compression 9

# 浅层推送（仅推送差异）
git push --no-verify

# 使用国内镜像
git remote add mirror https://gitee.com/yourname/TurboGitHub.git
git push mirror main
```
