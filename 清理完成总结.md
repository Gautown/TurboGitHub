## 🎯 清理方法

### 方法 1：使用清理工具（推荐）

```bash
# 运行清理工具
.\清理工具.bat

# 输出：
# [1/6] 清理 target 目录...
# [2/6] 清理 release 目录...
# [3/6] 清理日志文件...
# [4/6] 清理临时文件...
# [5/6] 清理系统文件...
# [6/6] 清理 Cargo 缓存...
# 清理完成！
```

### 方法 2：使用 cargo clean

```bash
# 清理构建产物
cargo clean

# 手动清理其他文件
rmdir /s /q release
del /q *.log *.tmp *.bak
```

### 方法 3：PowerShell 清理

```powershell
# 一键清理
Remove-Item -Recurse -Force target, release -ErrorAction SilentlyContinue
Get-ChildItem *.log, *.tmp, *.bak | Remove-Item -Force
```

## ⚠️ 注意事项

### 不要删除的文件

1. **Cargo.lock** - 锁定依赖版本
2. **配置文件** - `.cargo/config.toml`, `config.toml`
3. **源代码** - `core/src/`, `gui/src/`
4. **资源文件** - `assets/`
5. **许可证** - `LICENSE`
6. **文档** - `README.md`
7. **工具脚本** - `打包发布.bat`, `清理工具.bat`

### 可以安全删除的文件

1. **target/** - 构建产物（可重新编译）
2. **release/** - 发布包（可重新打包）
3. **\*.log** - 日志文件
4. **\*.tmp, \*.bak** - 临时和备份文件
5. **.DS_Store, Thumbs.db** - 系统文件

## 🛠️ 故障排除

### Q1: target 目录删除失败

**原因**: 文件被占用或损坏

**解决方案**:
```bash
# 1. 关闭占用进程
taskkill /F /IM cargo.exe
taskkill /F /IM rustc.exe

# 2. 使用清理工具
.\清理工具.bat

# 3. 重启电脑后手动删除
```

### Q2: cargo clean 报错

**原因**: target 目录损坏

**解决方案**:
```bash
# 使用清理工具
.\清理工具.bat

# 或强制删除
Remove-Item -Recurse -Force target -ErrorAction SilentlyContinue
```

### Q3: 清理后无法构建

**原因**: 清理过度或配置丢失

**解决方案**:
```bash
# 重新构建
cargo build --release

# 检查配置文件
dir .cargo\config.toml
dir config.toml
```
