## 🚀 发布流程

### 1. 本地测试发布
```bash
# 更新版本号
# 编辑 Cargo.toml, gui/Cargo.toml, release.toml

# 运行测试
cargo test

# 构建 Release
cargo build --release

# 本地测试
.\target\release\turbogithub-gui.exe

# 打包
.\打包发布.bat

# 测试安装包
installer_output\TurboGitHub-0.0.1-Setup.exe
```

### 2. 提交代码
```bash
git add -A
git commit -m "release: v0.0.1 首次发布"
```

### 3. 打标签
```bash
git tag -a v0.0.1 -m "TurboGitHub v0.0.1 首次发布"
```

### 4. 推送到 GitHub
```bash
git push TurboGitHub main
git push TurboGitHub v0.0.1
```

### 5. 创建 GitHub Release
1. 访问 https://github.com/Gautown/TurboGitHub/releases/new
2. 选择标签 v0.0.1
3. 填写发布说明（使用上面模板）
4. 上传构建的安装包
5. 点击 "Publish release"

### 6. 发布后验证
- [ ] Release 页面显示正常
- [ ] 所有下载链接有效
- [ ] SHA256 校验和正确
- [ ] 文档链接正确
- [ ] 标签正确

## 🛠️ 故障排除

### 问题 1: 构建失败
```bash
# 清理构建缓存
cargo clean

# 更新依赖
cargo update

# 重新构建
cargo build --release
```

### 问题 2: 打包失败
```bash
# 检查文件路径
dir target\release\turbogithub-gui.exe

# 手动创建目录
mkdir release\TurboGitHub-0.0.1

# 手动复制文件
copy target\release\turbogithub-gui.exe release\TurboGitHub-0.0.1\
```

### 问题 3: GitHub Actions 失败
- 检查 workflows 配置
- 查看构建日志
- 验证标签格式（v0.0.1）
- 检查 GITHUB_TOKEN 权限

### 问题 4: 安装包创建失败
- 检查 Inno Setup 版本
- 验证 installer.iss 语法
- 检查文件路径
- 以管理员身份运行

## 📈 最佳实践

### 1. 版本命名
- 格式：`v主版本。次版本.修订版`
- 示例：`v0.0.1`, `v1.2.3`
- 预发布：`v1.0.0-beta.1`

### 2. 提交信息
- 格式：`类型：描述`
- 示例：`release: v0.0.1 首次发布`
- 类型：`feat`, `fix`, `docs`, `style`, `refactor`, `test`, `chore`, `release`

### 3. 发布频率
- 小版本：每周或每两周
- 大版本：每月或每季度
- 紧急修复：及时发布

### 4. 文档维护
- 及时更新 CHANGELOG
- 保持 README 最新
- 添加使用示例
- 提供故障排除指南
