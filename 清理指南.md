# TurboGitHub 清理指南

## 🧹 清理目标

为打包发布做准备，清理冗余和临时文件。

---

## 📋 清理清单

### 1. 构建产物

#### 需要清理
- [x] `target/` - Rust 编译产物目录
  - 包含：调试和发布构建的所有文件
  - 大小：通常 2-10 GB
  - 清理方法：`cargo clean` 或删除目录

#### 保留文件
- [ ] `target/release/turbogithub-gui.exe` - 仅在打包前构建

### 2. 发布包

#### 需要清理
- [x] `release/` - 打包发布输出目录
  - 包含：之前发布的压缩包
  - 清理方法：直接删除

#### 保留策略
- 发布新版本前清理旧版本
- 或在 GitHub Releases 下载

### 3. 日志文件

#### 需要清理
- [x] `*.log` - 各种日志文件
  - 例如：`cargo.log`, `build.log`
  - 清理方法：`del *.log`

### 4. 临时文件

#### 需要清理
- [x] `*.tmp` - 临时文件
- [x] `*.bak` - 备份文件
- [x] `*.old` - 旧版本文件

### 5. 系统文件

#### 需要清理
- [x] `.DS_Store` - macOS 系统文件
- [x] `Thumbs.db` - Windows 缩略图缓存

### 6. IDE 文件

#### 需要清理
- [x] `.vscode/` - VS Code 配置（可选）
- [x] `.idea/` - IntelliJ 配置（可选）
- [x] `*.swp` - Vim 交换文件

---

## 🛠️ 清理方法

### 方法 1：使用清理脚本（推荐）

```bash
# 运行清理工具
.\清理工具.bat
```

**优点**：
- ✅ 一键清理所有文件
- ✅ 安全确认
- ✅ 详细输出
- ✅ 自动检查

### 方法 2：手动清理

```bash
# 1. 清理构建产物
cargo clean

# 2. 删除 release 目录
rmdir /s /q release

# 3. 删除日志文件
del /q *.log

# 4. 删除临时文件
del /q *.tmp *.bak *.old

# 5. 删除系统文件
del /q .DS_Store Thumbs.db
```

### 方法 3：PowerShell 清理

```powershell
# 一键清理命令
Remove-Item -Recurse -Force target, release -ErrorAction SilentlyContinue
Get-ChildItem *.log, *.tmp, *.bak, *.old | Remove-Item -Force
```

## ⚠️ 注意事项

### 不要删除的文件

1. **Cargo.lock**
   - 锁定依赖版本
   - 确保构建一致性
   - Git 提交需要

2. **配置文件**
   - `.cargo/config.toml`
   - `config.toml`
   - `release.toml`

3. **源代码**
   - `core/src/`
   - `gui/src/`

4. **资源文件**
   - `assets/`
   - `LICENSE`
   - `README.md`

5. **工具脚本**
   - `打包发布.bat`
   - `清理工具.bat`
   - `installer.iss`

### 谨慎删除的文件

1. **Cargo.lock**
   - 仅在更新依赖时删除
   - 让 Cargo 重新生成

2. **配置文件**
   - 确认有备份再删除
   - 或重命名为 `.bak`

## 🛠️ 故障排除

### Q1: target 目录删除失败

**原因**：文件被占用或损坏

**解决方案**：
```bash
# 1. 关闭所有占用进程
taskkill /F /IM cargo.exe
taskkill /F /IM rustc.exe

# 2. 使用清理工具
.\清理工具.bat

# 3. 重启电脑后手动删除
```

### Q2: cargo clean 失败

**原因**：target 目录损坏

**解决方案**：
```bash
# 强制删除
Remove-Item -Recurse -Force target -ErrorAction SilentlyContinue

# 或使用清理工具
.\清理工具.bat
```

### Q3: 清理后无法构建

**原因**：清理过度

**解决方案**：
```bash
# 重新构建
cargo build --release

# 如果需要调试版本
cargo build
```
