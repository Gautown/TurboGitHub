## 🎯 推荐的发布流程

### 方案 A：本地手动发布（适合首次）

#### 步骤 1：准备
```bash
# 1. 更新版本号
# 编辑 Cargo.toml, gui/Cargo.toml

# 2. 运行测试
cargo test
cargo fmt --check
cargo clippy -- -D warnings

# 3. 构建
cargo build --release
```

#### 步骤 2：打包
```bash
# 运行打包脚本
.\打包发布.bat

# 测试程序
.\release\TurboGitHub-0.0.1\turbogithub-gui.exe
```

#### 步骤 3：创建安装包（可选）
```bash
# 使用 Inno Setup
"C:\Program Files (x86)\Inno Setup 6\ISCC.exe" installer.iss
```

#### 步骤 4：发布
```bash
# 提交代码
git add -A
git commit -m "release: v0.0.1 首次发布"
git push TurboGitHub main

# 打标签
git tag -a v0.0.1 -m "TurboGitHub v0.0.1 首次发布"
git push TurboGitHub v0.0.1
```

#### 步骤 5：创建 GitHub Release
1. 访问 https://github.com/Gautown/TurboGitHub/releases/new
2. 选择标签 v0.0.1
3. 填写发布说明
4. 上传文件
5. 发布

---

### 方案 B：GitHub Actions 自动发布（推荐）

#### 步骤 1：准备和测试
```bash
# 本地测试
cargo test
cargo build --release
.\target\release\turbogithub-gui.exe
```

#### 步骤 2：推送标签
```bash
# 提交代码
git add -A
git commit -m "release: v0.0.1"
git push TurboGitHub main

# 打标签并推送
git tag -a v0.0.1 -m "TurboGitHub v0.0.1"
git push TurboGitHub v0.0.1
```

#### 步骤 3：等待自动发布
GitHub Actions 会自动：
1. 构建 Release 版本
2. 运行测试
3. 创建压缩包
4. 生成校验和
5. 创建 GitHub Release
6. 上传所有文件

#### 步骤 4：验证
访问 https://github.com/Gautown/TurboGitHub/releases 查看发布结果

## 📊 发布方案对比

| 方案 | 优点 | 缺点 | 推荐场景 |
|------|------|------|---------|
| **便携版** | 无需安装、即开即用 | 需要手动配置 | 技术用户、快速测试 |
| **Inno Setup** | 专业安装程序、支持卸载 | 需要额外工具 | 普通用户、正式发布 |
| **GitHub Actions** | 全自动、可重复 | 需要网络、配置复杂 | 持续发布、多平台 |
| **WiX Toolset** | 微软标准、企业级 | 学习曲线陡 | 企业部署 |
| **cargo-bundle** | 简单快速 | 功能较少 | 快速打包 |

## 🎯 首次发布推荐方案

### 组合方案（最佳用户体验）

**提供三种版本**：
1. **便携版** (.7z + .zip) - 技术用户
2. **安装包** (.exe) - 普通用户
3. **GitHub Release** - 自动发布

### 具体步骤

1. **本地打包**
   ```bash
   .\打包发布.bat
   ```

2. **创建安装包**
   ```bash
   "C:\Program Files (x86)\Inno Setup 6\ISCC.exe" installer.iss
   ```

3. **测试**
   - 测试便携版
   - 测试安装包
   - 测试卸载程序

4. **发布**
   ```bash
   git tag -a v0.0.1 -m "首次发布"
   git push TurboGitHub main
   git push TurboGitHub v0.0.1
   ```

5. **创建 Release**
   - 上传所有文件
   - 填写详细发布说明
   - 添加截图（可选）

## 📞 故障排除

### 常见问题

#### Q1: 打包脚本报错
```bash
# 检查 Rust 是否安装
rustc --version

# 检查构建是否成功
dir target\release\turbogithub-gui.exe

# 清理重新构建
cargo clean
cargo build --release
```

#### Q2: Inno Setup 编译失败
```bash
# 检查安装路径
dir "C:\Program Files (x86)\Inno Setup 6"

# 验证脚本语法
# 使用 Inno Setup IDE 打开 installer.iss 检查
```

#### Q3: GitHub Actions 失败
- 检查 workflows 配置
- 查看完整日志
- 验证标签格式（v0.0.1）
- 检查 GITHUB_TOKEN 权限

#### Q4: 文件太大
```bash
# 使用 UPX 压缩
upx --best --lzma target\release\turbogithub-gui.exe

# 检查依赖
cargo tree

# 移除不必要的依赖
```
