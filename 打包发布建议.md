# TurboGitHub 打包发布建议

## 📦 发布策略

### 推荐方案：使用 `cargo-binstall` + `Inno Setup` 创建 Windows 安装包

---

## 🎯 方案一：使用 cargo-bundle（简单快速）

### 1. 安装 cargo-bundle

```bash
cargo install cargo-bundle
```

### 2. 配置优化

您的 `gui/Cargo.toml` 已经配置了 `package.metadata.bundle`，很好！

### 3. 构建并发布

```bash
# 构建 Release 版本
cargo build --release

# 打包
cargo bundle --release
```

### 4. 输出位置

```
target/x86_64-pc-windows-msvc/release/bundle/
├── msi/           # Windows Installer 安装包
└── windows/       # 便携版目录
```

**优点**：
- ✅ 配置简单
- ✅ 自动生成 MSI 安装包
- ✅ 支持多平台

**缺点**：
- ⚠️ 自定义选项较少
- ⚠️ 功能相对基础

## 🎯 方案二：使用 Inno Setup（推荐）

### 1. 安装 Inno Setup

下载地址：https://jrsoftware.org/isdl.php

### 2. 创建安装脚本

创建文件 `installer.iss`：

```iss
#define MyAppName "TurboGitHub"
#define MyAppVersion "0.0.1"
#define MyAppPublisher "Gautown"
#define MyAppURL "https://github.com/Gautown/TurboGitHub"
#define MyAppExeName "turbogithub-gui.exe"

[Setup]
AppId={{A1B2C3D4-E5F6-7890-ABCD-EF1234567890}
AppName={#MyAppName}
AppVersion={#MyAppVersion}
AppPublisher={#MyAppPublisher}
AppPublisherURL={#MyAppURL}
AppSupportURL={#MyAppURL}
AppUpdatesURL={#MyAppURL}
DefaultDirName={autopf}\{#MyAppName}
DefaultGroupName={#MyAppName}
AllowNoIcons=yes
LicenseFile=LICENSE
OutputDir=installer_output
OutputBaseFilename=TurboGitHub-{#MyAppVersion}-Setup
SetupIconFile=assets\icons\logo.ico
Compression=lzma2/max
SolidCompression=yes
WizardStyle=modern
PrivilegesRequired=admin
ArchitecturesAllowed=x64
ArchitecturesInstallIn64BitMode=x64

[Languages]
Name: "chinesesimplified"; MessagesFile: "compiler:Languages\ChineseSimplified.isl"

[Tasks]
Name: "desktopicon"; Description: "{cm:CreateDesktopIcon}"; GroupDescription: "{cm:AdditionalIcons}"; Flags: unchecked
Name: "quicklaunchicon"; Description: "{cm:CreateQuickLaunchIcon}"; GroupDescription: "{cm:AdditionalIcons}"; Flags: unchecked; OnlyBelowVersion: 6.1; Check: not IsAdminInstallMode

[Files]
Source: "target\release\turbogithub-gui.exe"; DestDir: "{app}"; Flags: ignoreversion
Source: "config.toml"; DestDir: "{app}"; Flags: ignoreversion
Source: "core\config.toml"; DestDir: "{app}\core"; Flags: ignoreversion
Source: "assets\icons\logo.ico"; DestDir: "{app}\assets\icons"; Flags: ignoreversion
Source: "assets\images\*"; DestDir: "{app}\assets\images"; Flags: ignoreversion
Source: "启动 TurboGitHub.bat"; DestDir: "{app}"; Flags: ignoreversion
Source: "README.md"; DestDir: "{app}"; Flags: ignoreversion
Source: "LICENSE"; DestDir: "{app}"; Flags: ignoreversion

[Icons]
Name: "{group}\{#MyAppName}"; Filename: "{app}\{#MyAppExeName}"
Name: "{group}\{cm:UninstallProgram,{#MyAppName}}"; Filename: "{uninstallexe}"
Name: "{autodesktop}\{#MyAppName}"; Filename: "{app}\{#MyAppExeName}"; Tasks: desktopicon
Name: "{userappdata}\Microsoft\Internet Explorer\Quick Launch\{#MyAppName}"; Filename: "{app}\{#MyAppExeName}"; Tasks: quicklaunchicon

[Run]
Filename: "{app}\{#MyAppExeName}"; Description: "{cm:LaunchProgram,{#StringChange(MyAppName, '&', '&&')}}"; Flags: nowait postinstall skipifsilent

[Code]
function InitializeSetup(): Boolean;
var
  ResultCode: Integer;
begin
  Result := True;
  
  // 检查是否已安装
  if RegKeyExists(HKLM, 'Software\Microsoft\Windows\CurrentVersion\Uninstall\{#SetupSetting("AppId")}_is1') then
  begin
    if MsgBox('TurboGitHub 已经安装。是否要卸载旧版本并继续安装？', mbConfirmation, MB_YESNO or MB_DEFBUTTON2) = IDNO then
      Result := False;
  end;
end;
```

### 3. 构建步骤

```bash
# 1. 构建 Release 版本
cargo build --release

# 2. 使用 Inno Setup 编译安装包
# 在 Inno Setup IDE 中打开 installer.iss 并编译
# 或使用命令行：
"C:\Program Files (x86)\Inno Setup 6\ISCC.exe" installer.iss
```

### 4. 输出

生成 `installer_output/TurboGitHub-0.0.1-Setup.exe`

**优点**：
- ✅ 功能强大，高度可定制
- ✅ 支持卸载程序
- ✅ 支持中文界面
- ✅ 支持注册表操作
- ✅ 支持系统服务安装

**缺点**：
- ⚠️ 需要额外安装工具
- ⚠️ 配置稍复杂

## 🎯 方案三：使用 WiX Toolset（专业级）

### 1. 安装 WiX Toolset

```bash
# 使用 winget 安装
winget install WixToolset.WixToolset
```

### 2. 创建 WiX 配置文件

创建 `installer.wxs`：

```xml
<?xml version="1.0" encoding="UTF-8"?>
<Wix xmlns="http://schemas.microsoft.com/wix/2006/wi">
  <Product Id="*" Name="TurboGitHub" Language="1033" Version="0.0.1" Manufacturer="Gautown" UpgradeCode="12345678-1234-1234-1234-123456789012">
    <Package InstallerVersion="200" Compressed="yes" InstallScope="perMachine" />
    
    <MajorUpgrade DowngradeErrorMessage="A newer version of TurboGitHub is already installed." />
    
    <MediaTemplate EmbedCab="yes" />
    
    <Feature Id="ProductFeature" Title="TurboGitHub" Level="1">
      <ComponentGroupRef Id="ProductComponents" />
      <ComponentRef Id="ApplicationShortcut" />
    </Feature>
    
    <UI>
      <UIRef Id="WixUI_InstallDir" />
    </UI>
  </Product>

  <Fragment>
    <Directory Id="TARGETDIR" Name="SourceDir">
      <Directory Id="ProgramFilesFolder">
        <Directory Id="INSTALLFOLDER" Name="TurboGitHub" />
      </Directory>
      <Directory Id="ProgramMenuFolder">
        <Directory Id="ApplicationProgramsFolder" Name="TurboGitHub"/>
      </Directory>
    </Directory>
  </Fragment>

  <Fragment>
    <ComponentGroup Id="ProductComponents" Directory="INSTALLFOLDER">
      <Component Id="MainExecutable" Guid="*">
        <File Id="turbogithub.exe" Source="target\release\turbogithub-gui.exe" KeyPath="yes" />
      </Component>
      <Component Id="ConfigFile" Guid="*">
        <File Id="config.toml" Source="config.toml" KeyPath="yes" />
      </Component>
    </ComponentGroup>
    
    <Component Id="ApplicationShortcut" Directory="ApplicationProgramsFolder">
      <Shortcut Id="ApplicationStartMenuShortcut"
                Name="TurboGitHub"
                Description="GitHub 加速工具"
                Target="[INSTALLFOLDER]turbogithub-gui.exe"
                WorkingDirectory="INSTALLFOLDER"/>
      <RemoveFolder Id="ApplicationProgramsFolder" On="uninstall"/>
      <RegistryValue Root="HKCU"
                     Key="Software\Gautown\TurboGitHub"
                     Name="installed"
                     Type="integer"
                     Value="1"
                     KeyPath="yes"/>
    </Component>
  </Fragment>
</Wix>
```

### 3. 构建

```bash
# 编译 WiX 源文件
candle installer.wxs

# 链接生成 MSI
light installer.wixobj -o TurboGitHub-0.0.1.msi
```

**优点**：
- ✅ 微软官方工具
- ✅ 生成标准 MSI 安装包
- ✅ 适合企业部署

**缺点**：
- ⚠️ 学习曲线陡峭
- ⚠️ 配置复杂

## 🎯 方案四：创建便携版（Portable）

### 1. 构建 Release 版本

```bash
cargo build --release
```

### 2. 创建发布目录结构

```
TurboGitHub-0.0.1-portable/
├── turbogithub-gui.exe          # 主程序
├── config.toml                   # 配置文件
├── core/
│   └── config.toml              # 核心配置
├── assets/
│   ├── icons/                   # 图标资源
│   └── images/                  # 图片资源
├── 启动 TurboGitHub.bat         # 启动脚本
├── README.md                     # 说明文档
└── LICENSE                       # 许可证
```

### 3. 创建批处理脚本

创建 `打包发布.bat`：

```batch
@echo off
chcp 65001 >nul
echo ========================================
echo   TurboGitHub 打包发布工具
echo ========================================
echo.

set VERSION=0.0.1
set BUILD_DIR=target\release
set OUTPUT_DIR=release\TurboGitHub-%VERSION%

echo [1/4] 构建 Release 版本...
cargo build --release
if errorlevel 1 (
    echo [错误] 构建失败
    pause
    exit /b 1
)
echo [✓] 构建完成
echo.

echo [2/4] 创建发布目录...
if exist "%OUTPUT_DIR%" rmdir /s /q "%OUTPUT_DIR%"
mkdir "%OUTPUT_DIR%"
mkdir "%OUTPUT_DIR%\assets\icons"
mkdir "%OUTPUT_DIR%\assets\images"
echo [✓] 目录创建完成
echo.

echo [3/4] 复制文件...
copy "%BUILD_DIR%\turbogithub-gui.exe" "%OUTPUT_DIR%\"
copy "config.toml" "%OUTPUT_DIR%\"
copy "启动 TurboGitHub.bat" "%OUTPUT_DIR%\"
copy "README.md" "%OUTPUT_DIR%\"
copy "LICENSE" "%OUTPUT_DIR%\"
xcopy /E /I "assets\icons" "%OUTPUT_DIR%\assets\icons"
xcopy /E /I "assets\images" "%OUTPUT_DIR%\assets\images"
echo [✓] 文件复制完成
echo.

echo [4/4] 创建压缩包...
cd release
"C:\Program Files\7-Zip\7z.exe" a -t7z -mmt=on "TurboGitHub-%VERSION%-portable.7z" "TurboGitHub-%VERSION%"
"C:\Program Files\7-Zip\7z.exe" a -tzip -mmt=on "TurboGitHub-%VERSION%-portable.zip" "TurboGitHub-%VERSION%"
cd ..
echo [✓] 压缩包创建完成
echo.

echo ========================================
echo   发布完成！
echo ========================================
echo.
echo 输出文件:
echo   - release\TurboGitHub-%VERSION%-portable.7z
echo   - release\TurboGitHub-%VERSION%-portable.zip
echo   - release\TurboGitHub-%VERSION%\ (便携版目录)
echo.

pause
```

**优点**：
- ✅ 无需安装
- ✅ 即开即用
- ✅ 适合技术用户

**缺点**：
- ⚠️ 需要手动配置
- ⚠️ 不适合普通用户

## 🎯 方案五：使用 GitHub Actions 自动发布

### 创建 `.github/workflows/release.yml`

```yaml
name: Release

on:
  push:
    tags:
      - 'v*'

jobs:
  build-windows:
    runs-on: windows-latest
    
    steps:
    - uses: actions/checkout@v4
    
    - name: 安装 Rust
      uses: dtolnay/rust-toolchain@stable
      with:
        targets: x86_64-pc-windows-msvc
    
    - name: 缓存 Cargo 依赖
      uses: actions/cache@v4
      with:
        path: |
          ~/.cargo/registry
          ~/.cargo/git
          target
        key: ${{ runner.os }}-cargo-${{ hashFiles('**/Cargo.lock') }}
    
    - name: 构建 Release 版本
      run: cargo build --release --verbose
    
    - name: 创建发布包
      run: |
        mkdir release-package
        copy target\release\turbogithub-gui.exe release-package\
        copy config.toml release-package\
        copy 启动 TurboGitHub.bat release-package\
        copy README.md release-package\
        copy LICENSE release-package\
        xcopy /E /I assets release-package\assets
        
    - name: 创建压缩包
      run: |
        cd release-package
        7z a -tzip ../TurboGitHub-${{ github.ref_name }}-windows-x64.zip *
        
    - name: 上传发布资产
      uses: softprops/action-gh-release@v1
      with:
        files: TurboGitHub-*.zip
      env:
        GITHUB_TOKEN: ${{ secrets.GITHUB_TOKEN }}
```

**优点**：
- ✅ 自动化发布
- ✅ 每次 Tag 自动构建
- ✅ 生成多个平台包

**缺点**：
- ⚠️ 需要配置 GitHub Secrets
- ⚠️ 依赖网络

## 🎯 推荐发布流程

### 步骤 1：准备阶段

```bash
# 1. 更新版本号
# 编辑 Cargo.toml, gui/Cargo.toml, release.toml

# 2. 运行测试
cargo test

# 3. 构建 Release
cargo build --release

# 4. 本地测试
.\target\release\turbogithub-gui.exe
```

### 步骤 2：打包

**方案 A：便携版（推荐首发）**
```bash
# 运行打包脚本
.\打包发布.bat
```

**方案 B：Inno Setup 安装包**
```bash
# 使用 Inno Setup 编译
"C:\Program Files (x86)\Inno Setup 6\ISCC.exe" installer.iss
```

### 步骤 3：测试安装包

- [ ] 在干净系统上测试安装
- [ ] 测试程序运行
- [ ] 测试卸载程序
- [ ] 检查文件完整性

### 步骤 4：发布到 GitHub

```bash
# 1. 提交代码
git add -A
git commit -m "release: v0.0.1 首次发布"

# 2. 打标签
git tag -a v0.0.1 -m "TurboGitHub v0.0.1 首次发布"

# 3. 推送
git push TurboGitHub main
git push TurboGitHub v0.0.1
```

### 步骤 5：创建 GitHub Release

1. 访问 https://github.com/Gautown/TurboGitHub/releases
2. 点击 "Create a new release"
3. 选择标签 v0.0.1
4. 填写发布说明
5. 上传构建的安装包
6. 点击 "Publish release"
