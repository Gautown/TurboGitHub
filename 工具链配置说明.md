## 🎯 为什么使用 GNU 工具链

### GNU vs MSVC

| 特性 | GNU (MinGW) | MSVC |
|------|-------------|------|
| 编译器 | GCC | Microsoft C++ |
| 链接器 | ld | link.exe |
| 运行时 | MinGW-w64 | MSVCRT |
| 兼容性 | POSIX | Windows 原生 |
| 文件大小 | 较小 | 较大 |
| 启动速度 | 较快 | 较慢 |
| 调试支持 | GDB | Visual Studio |

### 项目选择 GNU 的原因

1. **跨平台一致性** - GNU 工具链在多个平台上行为一致
2. **更小的二进制文件** - 生成的可执行文件更小
3. **更快的编译速度** - 某些场景下编译更快
4. **更好的 POSIX 兼容性** - 适合 Rust 生态系统
5. **免费开源** - 完全开源，无许可限制

## 🛠️ 工具链管理命令

### 查看已安装工具链

```bash
rustup show
```

### 安装 GNU 工具链

```bash
rustup install stable-x86_64-pc-windows-gnu
```

### 设置默认工具链

```bash
rustup default stable-x86_64-pc-windows-gnu
```

### 卸载工具链

```bash
rustup toolchain uninstall <toolchain-name>
```

### 为项目指定工具链

```bash
# 在项目目录下运行
rustup override set stable-x86_64-pc-windows-gnu
```

### 查看项目工具链

```bash
rustup show active-toolchain
```

## 🔧 编译配置

### Cargo 配置

项目使用国内镜像源，配置文件位于 `.cargo/config.toml`：

```toml
[source.crates-io]
replace-with = 'ustc'

[source.ustc]
registry = "sparse+https://mirrors.ustc.edu.cn/crates.io-index/"

[build]
# 使用 GNU 工具链
target = "x86_64-pc-windows-gnu"

[target.x86_64-pc-windows-gnu]
# GNU 特定配置
linker = "x86_64-w64-mingw32-gcc"
```

### 编译命令

```bash
# 调试版本
cargo build

# Release 版本
cargo build --release

# 指定目标
cargo build --target x86_64-pc-windows-gnu

# 清理构建
cargo clean
```

## ⚠️ 常见问题

### Q1: 找不到 GNU 链接器

**错误信息**:
```
error: linker `x86_64-w64-mingw32-gcc` not found
```

**解决方案**:
1. 安装 MinGW-w64
2. 添加到系统 PATH
3. 或使用 rustup 安装的 GNU 工具链

### Q2: 编译时出现 MSVC 相关错误

**原因**: 可能使用了错误的工具链

**解决方案**:
```bash
# 检查当前工具链
rustup show

# 切换到 GNU
rustup default stable-x86_64-pc-windows-gnu

# 清理并重新编译
cargo clean
cargo build
```

### Q3: 如何完全清理 MSVC 工具链

**步骤**:
```bash
# 1. 终止所有 Rust 相关进程
taskkill /F /IM rustup.exe
taskkill /F /IM cargo.exe
taskkill /F /IM rustc.exe

# 2. 卸载 MSVC 工具链
rustup toolchain uninstall stable-x86_64-pc-windows-msvc
rustup toolchain uninstall 1.92.0-x86_64-pc-windows-msvc
rustup toolchain uninstall 1.93.0-x86_64-pc-windows-msvc

# 3. 验证
rustup show
```
